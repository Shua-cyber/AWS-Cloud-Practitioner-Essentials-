# Module 1: Introduction to the Cloud ☁️

## Overview
This module introduces **cloud computing fundamentals**, the **AWS Cloud value proposition**, and why organizations adopt the cloud.  
It lays the foundation for all later modules.

---

## Key Concepts

### What is Cloud Computing?
Cloud computing is the **on-demand delivery of IT resources** over the internet with **pay-as-you-go pricing**.  
Instead of owning physical servers or data centers, users can access compute, storage, and databases from AWS.

---

## Six Advantages of Cloud Computing
1. **Trade capital expense for variable expense** – No upfront hardware costs.  
2. **Benefit from massive economies of scale** – Lower pay-as-you-go prices due to AWS scale.  
3. **Stop guessing capacity** – Scale up/down based on demand.  
4. **Increase speed and agility** – Deploy new resources in minutes.  
5. **Stop spending on running and maintaining data centers** – Focus on innovation, not infrastructure.  
6. **Go global in minutes** – Deploy to multiple AWS Regions worldwide.

---

## Cloud Deployment Models
| Model | Description | Example Use Case |
|--------|--------------|------------------|
| **Cloud** | Fully hosted on AWS | Startups or new projects |
| **Hybrid** | Mix of on-premises and AWS resources | Gradual migrations |
| **On-Premises (Private Cloud)** | Self-managed infrastructure using virtualization | Highly regulated workloads |

---

## Cloud Service Models
| Model | Description | Example AWS Services |
|--------|--------------|----------------------|
| **IaaS** (Infrastructure as a Service) | Provides basic compute, storage, and networking building blocks | Amazon EC2, Amazon VPC |
| **PaaS** (Platform as a Service) | Provides a platform for developing and deploying applications | AWS Elastic Beanstalk |
| **SaaS** (Software as a Service) | Delivers software applications over the internet | Amazon Chime, AWS WorkMail |

---

## Shared Responsibility Model (Preview)
- **AWS:** Responsible for security **of** the cloud (infrastructure, hardware, networking).  
- **Customer:** Responsible for security **in** the cloud (data, configurations, IAM).

---

## AWS Global Infrastructure
- **Regions:** Physical locations worldwide containing multiple Availability Zones.  
- **Availability Zones (AZs):** Isolated data centers within a Region, designed for redundancy.  
- **Edge Locations:** Used by **Amazon CloudFront** to deliver content with low latency.

---

## Real-World Benefits
- Lower costs and faster innovation for startups and enterprises.  
- Scalability for variable workloads (e.g., retail traffic spikes).  
- Global reach for customers anywhere.

---

## Summary
Cloud computing delivers IT resources on demand with flexibility, scalability, and cost efficiency.  
AWS leads the market by providing global infrastructure, strong security, and a wide service portfolio.
